# 店小乖 - 需求设计文档

## 1. 系统角色

| 角色 | 终端 | 说明 |
|------|------|------|
| 顾客(C端) | 机器人平板 | 查看店铺信息、取号、注册/登录会员、领取眼镜 |
| 店员(B端) | 店家平板 | 录入店铺/商品数据、叫号、管理眼镜库存、查看会员信息 |
| 顾客(C端) | AR眼镜 | 玩游戏、知识问答、看视频、接收叫号通知 |

## 2. 核心业务流程

### 2.1 顾客端流程（机器人平板）

```
顾客靠近机器人
    ↓
浏览店铺宣传信息（从DB读取）
    ↓
输入手机号 → 注册/登录会员
  ├─ 新用户 → 自动注册，写入会员表
  └─ 老用户 → 查询已有记录，直接登录
    ↓
取号 → 生成排队号，关联会员
    ↓
领取眼镜 → 眼镜ID绑定手机号+排队号
    ↓
佩戴眼镜 → 玩游戏/问答/看视频（等待叫号）
```

### 2.2 店员端流程（店家平板）

```
店员录入/更新店铺信息、商品信息、优惠活动 → 写入DB
    ↓
查看排队列表 & 眼镜使用状态
    ↓
叫号 → 系统通过蓝牙通知对应眼镜
    ↓
顾客归还眼镜 → 解绑眼镜，回收入库
```

## 3. 核心功能清单

### 3.1 机器人平板（C端）

| 编号 | 功能 | 说明 |
|------|------|------|
| C-1 | 店铺信息展示 | 从DB读取并展示门店地址、营业时间、联系电话、门头照 |
| C-2 | 商品信息展示 | 展示商品列表（名称、图片、价格）、热销商品标识 |
| C-3 | 优惠活动展示 | 展示当前促销/优惠券信息 |
| C-4 | 排队情况展示 | 显示当前排队人数、预计等待时间 |
| C-5 | 会员注册/登录 | 输入手机号+验证码，新用户自动注册，老用户直接登录 |
| C-6 | 取号 | 登录后取号，生成排队号码，关联会员ID |
| C-7 | 领取眼镜 | 取号后分配一副空闲眼镜，绑定眼镜ID与用户手机号及排队号 |

### 3.2 店家平板（B端）

| 编号 | 功能 | 说明 |
|------|------|------|
| B-1 | 店铺信息管理 | 录入/编辑门店基本信息、图片、Logo |
| B-2 | 商品信息管理 | 增删改商品（名称、价格、图片、热销标记） |
| B-3 | 优惠活动管理 | 配置促销活动、优惠券信息 |
| B-4 | 排队叫号 | 查看排队列表，点击叫号 → 触发通知对应眼镜 |
| B-5 | 眼镜库存管理 | 查看所有眼镜状态（空闲/使用中）、绑定用户手机号、绑定时间 |
| B-6 | 眼镜解绑/回收 | 顾客归还后解除眼镜与用户的绑定 |
| B-7 | 会员数据查看 | 查看注册会员列表、游戏积分、奖励兑换状态 |
| B-8 | 样品管理 | 管理样品库存数量、领取记录 |

### 3.3 AR眼镜（C端）

| 编号 | 功能 | 说明 |
|------|------|------|
| G-1 | 叫号通知 | 距叫号还差5位时提醒；轮到时全屏弹窗+震动 |
| G-2 | 游戏 | 美食大挑战：声控跳跃游戏，积分达标即可获得奖励 |
| G-3 | 知识问答 | 门店相关趣味问答（备选） |
| G-4 | 视频播放 | 门店宣传视频（备选） |

## 4. 数据模型（核心表）

### 4.1 会员表 (member)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 会员ID |
| phone | VARCHAR(11) UK | 手机号（唯一键，用于判断是否已注册） |
| name | VARCHAR(50) | 会员姓名 |
| points | INT | 游戏积分 |
| created_at | DATETIME | 注册时间 |

> 注册逻辑：根据手机号查询，存在则直接返回（不INSERT），不存在则新增。

### 4.2 排队表 (queue)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 排队记录ID |
| queue_number | VARCHAR(10) | 排队号码（如 A001） |
| member_id | BIGINT FK | 关联会员ID |
| glasses_id | BIGINT FK | 关联眼镜ID（可为空，领取眼镜后绑定） |
| status | ENUM | waiting / called / completed / cancelled |
| created_at | DATETIME | 取号时间 |
| called_at | DATETIME | 叫号时间 |

### 4.3 眼镜表 (glasses)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 眼镜ID |
| device_code | VARCHAR(50) UK | 设备编号（蓝牙标识） |
| status | ENUM | available / in_use / charging / offline |
| current_member_id | BIGINT FK | 当前使用者会员ID（空闲时为NULL） |
| current_queue_id | BIGINT FK | 当前关联的排队记录ID |
| bound_at | DATETIME | 绑定时间 |

> 核心关系：叫号时，通过 queue.glasses_id 找到对应眼镜，推送通知。

### 4.4 店铺信息表 (store)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 店铺ID |
| name | VARCHAR(100) | 店铺名称 |
| address | VARCHAR(200) | 地址 |
| phone | VARCHAR(20) | 联系电话 |
| business_hours | VARCHAR(100) | 营业时间 |
| logo_url | VARCHAR(500) | Logo图片URL |
| cover_url | VARCHAR(500) | 门头封面图URL |

### 4.5 商品表 (product)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 商品ID |
| store_id | BIGINT FK | 所属店铺 |
| name | VARCHAR(100) | 商品名称 |
| price | DECIMAL(10,2) | 原价 |
| sale_price | DECIMAL(10,2) | 销售价 |
| image_url | VARCHAR(500) | 商品图片 |
| is_hot | BOOLEAN | 是否热销 |
| sales_7d | INT | 近7天销量 |

### 4.6 样品管理表 (sample)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 记录ID |
| store_id | BIGINT FK | 所属店铺 |
| product_name | VARCHAR(100) | 样品名称 |
| total_count | INT | 样品总数 |
| remaining_count | INT | 剩余数量 |

### 4.7 样品领取记录表 (sample_record)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 记录ID |
| member_id | BIGINT FK | 领取会员 |
| sample_id | BIGINT FK | 样品ID |
| created_at | DATETIME | 领取时间 |

> 规则：同一手机号对同一样品仅可领取1次。

### 4.8 游戏记录表 (game_record)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 记录ID |
| member_id | BIGINT FK | 玩家会员ID |
| glasses_id | BIGINT FK | 使用眼镜ID |
| score | INT | 游戏得分 |
| reward_threshold | INT | 达标分数线（由店家配置） |
| reward_earned | BOOLEAN | 是否达标获得奖励 |
| reward_claimed | BOOLEAN | 奖品是否已兑换 |
| created_at | DATETIME | 游戏时间 |

## 5. 核心交互链路

```
┌──────────────┐         ┌──────────┐         ┌──────────────┐
│  机器人平板   │◄──读取──│    DB    │──写入──►│  店家平板     │
│  (C端展示)   │──写入──►│          │◄──读取──│  (B端管理)   │
└──────┬───────┘         └────┬─────┘         └──────┬───────┘
       │                      │                       │
       │ 领取眼镜              │ 叫号数据               │ 叫号操作
       ▼                      ▼                       │
┌──────────────┐         ┌──────────┐                │
│   AR眼镜     │◄─蓝牙──│  店家平板  │◄───────────────┘
│  (游戏/通知) │         │  (蓝牙网关)│
└──────────────┘         └──────────┘
```

**叫号通知链路：**
店员点击叫号 → 更新queue表status → 查询queue.glasses_id → 通过蓝牙向对应眼镜推送通知

## 6. 关键业务规则

1. **会员注册**：手机号为唯一键，已注册用户直接返回，不重复创建
2. **取号限制**：同一手机号每日最多取3次
3. **样品领取**：同一手机号对同一样品仅可领取1次
4. **眼镜绑定**：一副眼镜同一时刻只能绑定一个用户；用户归还后解绑
5. **叫号提醒**：距目标号差5位时第一次提醒；轮到时第二次提醒（全屏+震动）
6. **游戏奖励**：积分达到店家设定的分数线即可获得奖励
