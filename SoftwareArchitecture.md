## 二、核心算法/功能模块

### 模块总览

```
┌─────────────────────────────────────────────────────────────┐
│                        店小乖系统                            │
├──────────────┬──────────────┬───────────────┬───────────────┤
│   会员模块    │   排队叫号模块 │   眼镜管理模块  │  内容管理模块   │
│  Member      │  Queue       │  Glasses      │  Store/Product│
├──────────────┼──────────────┼───────────────┼───────────────┤
│ · 手机号注册  │ · 自增取号    │ · 设备绑定/解绑 │ · 店铺信息CRUD │
│ · 幂等登录    │ · 状态流转    │ · 库存状态管理  │ · 商品CRUD    │
│ · 积分管理    │ · 每日限频    │ · 实时占用查询  │ · 样品领取    │
│              │ · 叫号通知    │               │ · 游戏记录    │
└──────────────┴──────────────┴───────────────┴───────────────┘
```

### 模块详细说明

#### 1. 会员模块（Member）

| 功能 | 算法/逻辑 |
|------|----------|
| 注册/登录 | **幂等注册**：`SELECT` by phone → 存在则直接返回，不存在则 `INSERT` 后返回。一个接口同时完成注册和登录 |
| 唯一性保证 | phone 字段设 `UNIQUE` 约束，数据库层兜底防重 |

#### 2. 排队叫号模块（Queue）

| 功能 | 算法/逻辑 |
|------|----------|
| 自增取号 | 查询当日最大 queue_number，+1 生成新号（格式 `A001`）。无记录时从 `A001` 开始 |
| 每日限频 | `SELECT COUNT` 当日该 member_id 的取号次数，≥3 则拒绝 |
| 状态流转 | `waiting` → `called` → `completed`；可从 `waiting` 直接 → `cancelled` |
| 预估等待 | 当前 waiting 人数 × 单均就餐时长（配置值，默认15分钟） |
| 叫号通知 | 叫号时更新 status + called_at，前端轮询或蓝牙推送至眼镜 |

```
状态机：
  waiting ──叫号──→ called ──就餐完成──→ completed
     │
     └──取消──→ cancelled
```

#### 3. 眼镜管理模块（Glasses）

| 功能 | 算法/逻辑 |
|------|----------|
| 绑定 | 校验眼镜 status == `available` → 更新 current_member_id / current_queue_id / status = `in_use` / bound_at |
| 解绑 | 清空 current_member_id / current_queue_id → status = `available` / bound_at = NULL |
| 库存概览 | `GROUP BY status` 统计各状态数量，B端看板展示 |

```
状态机：
  available ──绑定──→ in_use ──归还解绑──→ available
      │                                      ▲
      └──充电/离线──→ charging / offline ─────┘
```

#### 4. 内容管理模块（Store / Product / Sample / Game）

| 功能 | 算法/逻辑 |
|------|----------|
| 店铺信息 | 单商家模式，全局仅一条 store 记录，GET 取 / PUT 改 |
| 商品管理 | 标准 CRUD；`is_hot` 字段标记热销，`?hot=true` 筛选 |
| 样品领取 | 领取前校验：① remaining_count > 0 ② 该 member 未领过该 sample（查 sample_record 去重）；通过后 remaining_count - 1 + 写 record |
| 游戏积分 | 提交 score，后端对比 reward_threshold：score ≥ 阈值 → reward_earned = true |

#### 5. 看板聚合模块（Dashboard）

| 指标 | 计算方式 |
|------|---------|
| 今日新增会员 | `SELECT COUNT` from member WHERE created_at = today |
| 今日样品领取 | `SELECT COUNT` from sample_record WHERE created_at = today |
| 当前排队人数 | `SELECT COUNT` from queue WHERE status = 'waiting' |
| 眼镜使用中 | `SELECT COUNT` from glasses WHERE status = 'in_use' |
| 累计会员总数 | `SELECT COUNT` from member |

## 三、第三方库/服务

### 后端（Python）

| 库 | 版本 | 用途 |
|----|------|------|
| **FastAPI** | 0.110+ | Web 框架，自动生成 OpenAPI 文档 |
| **Uvicorn** | 0.29+ | ASGI 服务器，运行 FastAPI |
| **SQLAlchemy** | 2.0+ | ORM，数据库模型定义与查询 |
| **Pydantic** | 2.0+ | 请求/响应数据校验（FastAPI 内置） |
| **python-multipart** | 0.0.9+ | 支持表单文件上传（商品图片等） |

> 数据库使用 Python 内置 sqlite3 驱动，无需额外安装。

### 前端（TypeScript / React）

| 库 | 版本 | 用途 |
|----|------|------|
| **React** | 18.x | UI 框架 |
| **React Router** | 6.x | SPA 路由管理 |
| **Ant Design** | 5.x | UI 组件库（表格、表单、按钮、卡片、菜单等） |
| **Axios** | 1.x | HTTP 请求客户端 |
| **Vite** | 5.x | 前端构建工具 |
| **TypeScript** | 5.x | 类型安全 |

### 外部服务

| 服务 | 用途 | 备注 |
|------|------|------|
| 无 | - | Demo 阶段不依赖任何外部云服务，全部本地运行 |

> 说明：本项目为黑客松 Demo，所有功能本地自闭环，不依赖短信网关、云存储、第三方登录等外部服务。验证码环节 Demo 中跳过或使用固定验证码。

### 2. C端核心流程（顾客取号领眼镜）

```
顾客                    机器人平板(React)              FastAPI               SQLite
 │                          │                          │                      │
 │──── 靠近机器人 ──────────→│                          │                      │
 │                          │── GET /api/store ────────→│── SELECT store ─────→│
 │                          │←── 店铺信息 ──────────────│←── store data ───────│
 │←── 展示首页+店铺信息 ─────│                          │                      │
 │                          │                          │                      │
 │──── 点击「取号」──────────→│                          │                      │
 │                          │                          │                      │
 │──── 输入手机号 ───────────→│                          │                      │
 │                          │── POST /members/register─→│── SELECT by phone ──→│
 │                          │                          │   存在→返回 / 不存在→INSERT
 │                          │←── member 对象 ───────────│←─────────────────────│
 │                          │                          │                      │
 │                          │── POST /queue/take-number→│── 校验每日≤3次 ──────→│
 │                          │                          │── INSERT queue ──────→│
 │                          │←── queue (A001) ──────────│←─────────────────────│
 │                          │                          │                      │
 │                          │── POST /glasses/{id}/bind→│── UPDATE glasses ────→│
 │                          │←── glasses 绑定成功 ──────│←─────────────────────│
 │                          │                          │                      │
 │←── 展示排队号+眼镜信息 ───│                          │                      │
 │                          │                          │                      │
 │──── 领取眼镜，开始游戏 ──→│                          │                      │
```

### 3. B端核心流程（店员叫号管理）

```
店员                    店家平板(React)               FastAPI               SQLite
 │                          │                          │                      │
 │──── 打开排队管理 ─────────→│                          │                      │
 │                          │── GET /queue/list ───────→│── SELECT queues ────→│
 │                          │←── 排队列表 ──────────────│←─────────────────────│
 │←── 展示排队列表 ──────────│                          │                      │
 │                          │                          │                      │
 │──── 点击「叫号」──────────→│                          │                      │
 │                          │── POST /queue/{id}/call ─→│── UPDATE status ────→│
 │                          │                          │   = 'called'          │
 │                          │                          │── 查询关联 glasses ──→│
 │                          │←── 叫号成功 ──────────────│←─────────────────────│
 │                          │                          │                      │
 │                          │          ┌───── 蓝牙通知 ─────→ AR眼镜弹窗提醒    │
 │                          │          │               │                      │
 │←── 状态更新为「已叫号」───│          │               │                      │
 │                          │                          │                      │
 │──── 顾客就餐完归还眼镜 ──→│                          │                      │
 │                          │── POST /glasses/{id}/unbind→│── UPDATE glasses ──→│
 │                          │                          │   status='available'  │

 │                          │                          │   status='completed'  │
 │                          │←── 操作成功 ──────────────│←─────────────────────│
 │←── 眼镜已回收 ───────────│                          │                      │
```
